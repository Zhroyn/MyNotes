### 寻址方式
- 立即寻址方式
  - `mov ax, 1234h`
- 寄存器寻址方式
  - `mov ax, bx`
- 直接寻址方式
  - 段地址默认在DS中，操作数大小由其他操作对象（如寄存器）决定
  - `mov ax, [1234h]`
  - `mov es:[5678h], bl`
- 寄存器间接寻址方式
  - 只允许使用 `bx` `bp` `si` `di`
  - 若使用 `bx` `si` `di`，则默认以 `ds` 为段地址
  - 若使用 `bp`，则默认以 `ss` 为段地址
  - `mov ax, cs:[bx]`
  - `mov [bp], cv`
- 寄存器相对寻址方式
  - 若使用 `bx` `si` `di`，则默认以 `ds` 为段地址
  - 若使用 `bp`，则默认以 `ss` 为段地址
  - `mov ax, [di+1234h]`
- 基址变址寻址方式
  - 操作数的有效地址由基址寄存器之一的内容与变址寄存器之一的内容相加得到，即 `[bx/bp + si/di]`
  - 若使用 `bp`，则默认以 `ss` 为段地址，否则以 `ds` 为段地址
  - `mov ax, es:[bx+si]`
- 相对基址变址寻址方式
  - 操作数的有效地址由基址寄存器之一的内容与变址寄存器之一的内容及给定的8位或16位位位移量相加得到
  - `mov ax, [bx+di+1234h]`
  - `mov ax, 1234h[bx+di]`
  - `mov ax, 1234h[bx][di]`
  - `mov ax, 1234h[di][bx]`








### 寄存器
#### 通用寄存器
- `ax` : `ah`, `al`
  - 累加寄存器，运算时较多使用
- `bx` : `bh`, `bl`
  - 基址寄存器，经常用于存放一段内存的偏移地址
- `cx` : `ch`, `cl`
  - 计数寄存器，经常用于存放重复操作的次数
- `dx` : `dh`, `dl`
  - 数据寄存器，有时存放32位数据的高16位

#### 段寄存器
- `cs` : 代码段寄存器，用于存放当前正在执行的程序段的段地址
- `ds` : 数据段寄存器，用于存放数据段的段地址
- `ss` : 堆栈段寄存器，用于存放堆栈段的段地址
- `es` : 附加段寄存器，用于存放另一个数据段的段地址
- 8086cpu不支持将数据直接送入段寄存器，所以需要借助其他寄存器或内存单元改变其中数据

#### 地址寄存器
- `bp` : 基址指针，用于存放内存中数据的偏移地址
- `sp` : 堆栈指针，用于存放堆栈栈顶的偏移地址
- `si` : 源变址寄存器，用于存放内存中源数据区的偏移地址，所谓变址寄存器，是指在某些指令作用下它可以自动地递增或递减其中的值
- `di` : 目的变址寄存器，用于存放内存中目的数据区的偏移地址，并在某些指令作用下可以自动地递增或递减其中的值

#### 指令指针寄存器
- `ip` : 指令指针寄存器，用于存放即将执行指令的偏移地址

#### 标志寄存器
- `FLAGS` : 存放CPU的两类标志
  - `OF` : 溢出标志。`OF=l` 表示两个有符号数的运算结果超出了可以表示的范围
  - `DF` : 方向标志。`DF=0` 时，每次执行字符串指令后，源或目的地址指针用加法自动修改地址；`DF=l`时用减法来修改地址。它用来控制地址的变化方向
  - `IF` : 中断允许标志。`IF=l` 表示允许处理器响应可屏蔽中断请求信号，称为开中断，`IF=0` 表示不允许处理器响应可屏蔽中断请求信号，称为关中断
  - `SF` : 符号标志。`SF=1` 表示运算结果为负，`SF=0` 表示运算结果为非负
  - `ZF` : 零标志。`ZF=l` 表示运算结果为零
  - `AF` : 辅助进位标志。在进行字操作时，低字节向高字节进位时，AF=1，否则为0。一般用于两个BCD数运算后调整结果用，对其他数的运算没有意义。
  - `PF` : 奇偶标志。`PF=1` 表示运算结果的低8位中有偶数个1，`PF=0` 表示有奇数个1
  - `CF` : 进位/借位标志。`CF=1` 表示两个无符号数的加法运算有进位，或者是减法运算有借位，需要对它们的高位进行补充处理；`CF=0` 表示没有产生进位或借位。同样，进行有符号数运算时也会产生新的CF标志，此时程序员可以不关心CF标志。









### 指令
#### 算术运算指令
- `add a,b` 使a中的值加上b或b中的值
- `sub a,b` 使a中的值减去b或b中的值
- `mul/imul a` 非符号数/有符号数乘法
  - 当操作数为8位时，`ax = al * a`
  - 当操作数为16位时，`dx:ax = ax * a`
  - 当操作数为32位时，`edx:eax = eax * a`
- `div/idiv a` 非符号数/有符号数除法
  - 当操作数是8位时，`ax % a`，商在 `al` 中，余数在 `ah` 中
  - 当操作数是16位时，`dx:ax % a`，商在 `ax` 中，余数在 `dx` 中
  - 当操作数是32位时，`edx:eax % a`，商在 `eax` 中，余数在 `edx` 中
- `cmp a,b` 通过做减法运算影响标志寄存器
  - 若 `a=b` 则 `ZF=1`，若 `a≠b` 则 `ZF=0`
  - 若 `a<b` 则 `CF=1`，若 `a≥b` 则 `CF=0`
- `inc a` 自加1
- `dec a` 自减1
- `neg a` 取反

#### 传送指令
- `mov a,b` 将b或b中的值送入a中
- `xchg a,b` 交换 `a` 和 `b` 的值，必须是 `ax` `bx` `cx` `dx` `si` `di` 或内存，不能同时是内存
- `push a` 将寄存器a中的数据送入栈
- `pop a` 从栈顶取出数据送入寄存器a
  - 8086cpu的入栈和出栈操作都是以字为单位进行的
  - 入栈时，字从栈顶开始自高向低存放，其中高地址单元存放高8位，低地址单元存放低8位
  - 任意时刻，`ss:sp`指向栈顶元素，`push`执行以后`sp`会减2

#### 转移指令
- 无条件转移指令
  - `jmp a:b` 用a修改CS，用b修改IP
  - `jmp a` 用a或a中的值修改IP
  - `jmp label` 用标号的CS和IP修改CS和IP
    - `CS`为代码段寄存器，`IP`为指令指针寄存器
    - 8086机中，任意时刻，CPU将`CS:IP`指向的内容当作指令执行
    - 读取一条指令后，`IP`会加上所读取指令的长度
- 条件转移指令
  - `je/jz label` : 相等/结果为零，条件为 `ZF=1`
  - `jne/jnz label` : 不相等/结果不为零，条件为 `ZF=0`
  - `js label` : 结果为负，条件为 `SF=1`
  - `jns label` : 结果非负，条件为 `SF=0`
  - `jb/jnae/jc label` : 小于/不大于等于/有进位，条件为 `CF=1`
  - `jnb/jae/jnc label` : 不小于/大于等于/无进位，条件为 `CF=0`
  - `jna/jbe label` : 不大于/小于等于，条件为 `CF=1 or ZF=1`
  - `ja/jnbe label` : 大于/不小于等于，条件为 `CF=0 and ZF=0`
- 循环指令
  - `loop label` : 将 `(cx)` 减一后，若 `(cx)` 不为零则转移
- 过程指令
  - `call label` : 将当前的CS和IP压入栈中，然后转移
  - `ret` 用栈中的数据，修改IP的内容，从而实现近转移
  - `retf` 用栈中的数据，修改CS和IP中的内容，从而实现远转移
- 中断指令
  - `int x` 利用中断类型码，引发中断过程
    - `int 21h` 根据AH中的值，进行DOS系统功能调用
      - `AH=4C` : 带返回码结束，`AL=返回码`
      - `AH=09` : 显示字符串，`DS:DX=串地址`，`$` 结束字符串
      - `AH=01` : 键盘输入并回显，`AL=输入字符`
      - `AH=02` : 显示输出，`DL=输出字符`

#### 逻辑指令
- `and a,b` : `a = a & b`
- `or a,b` : `a = a | b`
- `xor a,b` : `a = a ^ b`
- `not a`  : `a = ~a`
- `test a,b` 在两个操作数的对应位之间进行 AND 操作, AND 指令唯一不同的地方是，TEST 指令不修改目标操作数
- `shl a,b` 逻辑左移指令(left)，最低位用0补充
- `shr a,b` 逻辑右移指令(right)，最高位用0补充
- `sal a,b` 算数左移指令，在执行时将操作数看成带符号数进行移位
- `sar a,b` 算数右移指令，在执行时将操作数看成带符号数进行移位
- `rol a,b` 循环左移
- `ror a,b` 循环右移







### 伪指令
#### 数据定义伪指令
- `[变量名] db/dw/dd/dq/dt 表达式 [注释]`
  - `db` : 定义字节型变量，每变量分配1个存储单元
  - `dw` : 定义字型变量，每变量分配2个存储单元
  - `dd` : 定义双字型变量，每变量分配4个存储单元
  - `dq` : 定义四字型变量，每变量分配8个存储单元
  - `dt` : 定义十字型变量，每变量分配10个存储单元
- `n dup(表达式)` 将表达式重复 `n` 次

#### 运算符伪指令
- `seg x` 获得变量或标号的段地址，编译时会会把 `seg a` 编译成 `seg a - 首段段地址`
- `offset x` 获得变量或标号的偏移地址

#### 属性说明伪指令
- `类型 ptr 地址表达式` 确定地址表达式的存储单元为指定的类型，即用在地址表达式之前，用于指定或临时改变变量名和标号名的类型

#### 段定义与段分配伪指令
- `x segment` 说明一个段开始
- `x ends` 说明一个段结束
- `assume reg:seg,[reg:seg,...]` 建立段和段寄存器之间的联系，放在代码段内和段定义语句之后

#### 程序模块命名与程序结束伪指令
- `end [起始地址标号]` 表示程序结束，后为主模块起始地址，汇编时会将起始地址标号的段地址和偏移地址赋给CS和IP

### Turbo Debugger 常用快捷键
- `F2` 设置断点
- `F4` 运行至光标
- `F7` 单步跟踪
- `F8` 单步跳过
- `F9` 运行程序
- `Ctrl + F2` 重载程序
<br>

- `Ctrl + G` 跳转到指定地址
- `Ctrl + O` 跳转回程序执行处
- `Alt + x` 退出调试器






### DosBox 调试指令
- `-t` 单步执行
- `-p` 以汇编指令的格式在内存中写入一条机器指令
- `-q` 退出
<BR>

- `-r` 查看所有寄存器的值
- `-r reg` 修改寄存器中的值
<BR>

- `-d` 从当前或之前查看位置开始查看内存
- `-d addr` 从指定位置开始查看内存
- `-d addr n` 从指定位置开始查看内存，到指定偏移地址结束
<BR>

- `-e addr` 从指定位置开始修改内存中的内容，按空格进入下一个，按eNTER结束修改
- `-e addr a b c ...` 从指定位置开始，用输入内容修改内存中的内容
<BR>

- `-a` 以汇编指令的格式，从当前位置开始写入机器指令
- `-a addr` 以汇编指令的格式，从指定位置开始写入机器指令
- `-u addr` 将内存中的机器指令翻译成汇编指令
